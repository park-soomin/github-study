    - git 유저 이름 세팅
        git config --global user.email "soomini030113@gmail.com"

- 2강
    - git에 저장
        command + s                 // 저장
        git add 파일명                // staging
        git commit -m '아무메세지'     // repository에 저장

    - 다른 기능들
        git add 파일명1 파일명2     // 동시에 여러개 staging
        git add .                // 모든 파일 전부 staging
        git status               // 변경된 파일, staging된 파일 모두 알려줌(상태창)
        git log --all --oneline  // commit 기록 보여줌

    - commit은 언제?
        간단한 기능 추가할 때마다 하는 것이 best


- 3강
    - commit을 더 쉽게
        왼쪽 git처럼 생긴 메뉴로 들어가기
        변경사항 생기면 옆에 자동으로 뜸
        + 누르면 add
        체크마크 누르면 git commit
        (파일 많고 복잡하면 이 방법이 편함)

    - git diff 
        바로 전 commit과 현재 코드 차이점 비교해 줌
        
        - 부가 기능
            git diff 커밋id                // 과거의 특정 commit과 현재 파일 비교
            git diff 커밋id1 커밋id2        // 과거의 특정 commit 2개 차이점 비교
            git difftool                  // 비주얼적으로 좋음
            git graph와 같은 부가기능 사용하면 더 편리함


- 4강
    - branch
        프로젝트의 복사본 만들어서 branch에 먼저 개발

    - branch 생성
        - git branch 브랜치이름
            프로젝트 사본 생성
        
        - git switch 브랜치이름
            git branch로 만든 브랜치로 이동
            이동하면 여기서 개발 및 commit 마음대로 할 수 있음
        
        - git log --graph --oneline --all
            branch와 commit 내역을 한눈에 그래프로 볼 수 있음

        - 브랜치 병합
            git switch main
            git merge 브랜치이름

        - branch conflict
            master 브랜치와 새로운 branch에서 같은 파일과 같은 줄 수정하면 merge conflict 발생
            어떤 파일 남길지 결정했으면
            git add 파일명
            git commit -m '메세지'


- 5강
    - 3-way merge
        merge 명령을 내리면 두 브랜치의 코드르 합쳐서 새로운 commit 자동 생성
        merge의 기본 동작 방식

    - fast-forward merge
        새로운 브랜치에만 commit, 기준 브랜치에는 신규 commit X
        딱히 합칠게 없으니 신규 브랜치한테 "지금부터 니 이름은 main 브랜치여" 하는 것

    - 브랜치 삭제
        git branch -d 브랜치이름    // 병합이 완료된 브랜치 삭제시
        git branch -D 브랜치이름    // 병합하진 않은 브랜치 삭제시

    - rebase and merge
        - rebase
            브랜치의 시작점을 다른 commit으로 옮겨주는 행위
        
        1. rebase를 이용하여 신규 브랜치의 시작점을 main 브랜치 최근 commit으로 옮김
        2. fast-forward merge 하는 것

        git switch 새로운브랜치
        git rebase main
        git switch main
        git merge 새로운브랜치

    - squash and merge

        - 사용 이유
            모든 브랜치를 3-way merge하면 브랜치가 많아졌을 때 로그가 매우 복잡해보임
            3-way merge된 브랜치들의 commit 내역도 다 같이 출력되어 더러워짐

        - 동작 방식
            새로운 브랜치에 있던 commit들 연결이 아닌, 똑 떼와서 main 브랜치에 붙임 (텔레포트)

            git switch main
            git merge --squash 브랜치이름
            git commit -m '메세지'


- 6강
    - git restore
        - git restore 파일명
            최근 commit된 상태로 돌릴 수 있음
        - git restore --source 커밋아이디 파일명
            입력된 파일이 특정 커밋아이디 시점으로 돌아감
        - git restore --staged 파일명
            특정 파일 staging 취소

    - git revert
        - git revert 커밋아이디
            그 커밋아이디에서 일어난 일만 취소
            실행 시 에디터에 커밋메세지 수정 후 닫기
        
    - git reset
        - git reset --hard 커밋아이디
            그 커밋 생성될때의 시간으로 돌려줌
            프로젝트 망하거나 짧은 거리 되돌아갈때만 사용
        - git reset --soft 커밋아이디
            커밋아이디가 staging area에 남아있음
        - git reset --mixed 커밋아이디
            커밋아이디가 staging 되지 않은 상태
            git add, commit 할 수 있음


- 7강
    - repository
        git이 파일 버전을 저장해 두는 곳
        로컬 작업 폴더에는 .git이라는 파일이 있음 <- 이게 repository
        실제로 개발할 때는 온라인 repository 많이 사용

    1. 작업폴더 생성, 터미널에서 git init
    2. git branch -M main       // 기본 브랜치 이름 main으로 변경
    3. commit
    4. git push -u 원격저장소주소 main

    - 원격저장소 길게 입력 안하는 방법
        git remote add origin(변수이름) 원격저장소주소
        이후부터는 git push -u origin main 으로 사용 가능
        (참고) -u 는 방금 입력한 주소 기억하라는 뜻
              한번 쓰면 다음부터는 git push만 하면 됨

    - 원격저장소에 있던 거 그대로 내려받기
        git clone 원격저장소주소


- 8강
    - 협업
        git clone 원격저장소주소
        이러면 원격저장소에 있는 소스코드가 그대로 내려받아짐
        팀원이 git push 하려면 setting -> collaborators에 등록되어 있어야 함

        - 문제 발생
            갑자기 다른 놈이 만든 파일이 원격저장소에 생기면 git push 못함
            원격저장소가 타인에 의해 업데이트가 되면 git pull 원격저장소주소 하고 그대로 git push 하면 됨

    - (참고)
        git pull은 git fetch + git merge임
        git fetch는 원격저장소에 있는 commit중에 로컬이 없는 신규 commit 가져오라는 명령어
        git merge는 그걸 merge하라는 명령어
        git pull 할 때 2명이 같은 파일 건드리면 merge conflict 날 수 있음
