- 1강
    - git 사용 이유
        파일 내용 보존 및 관리

    - git 유저 이름 세팅
        git config --global user.email "soomini030113@gmail.com"
        git config --global user.name "박수민"


- 2강
    - git에 저장
        command + s                 // 저장
        git add 파일명                // staging
        git commit -m '아무메세지'     // repository에 저장

    - 다른 기능들
        git add 파일명1 파일명2     // 동시에 여러개 staging
        git add .                // 모든 파일 전부 staging
        git status               // 변경된 파일, staging된 파일 모두 알려줌(상태창)
        git log --all --oneline  // commit 기록 보여줌

    - commit은 언제?
        간단한 기능 추가할 때마다 하는 것이 best


- 3강
    - commit을 더 쉽게
        왼쪽 git처럼 생긴 메뉴로 들어가기
        변경사항 생기면 옆에 자동으로 뜸
        + 누르면 add
        체크마크 누르면 git commit
        (파일 많고 복잡하면 이 방법이 편함)

    - git diff 
        바로 전 commit과 현재 코드 차이점 비교해 줌
        
        - 부가 기능
            git diff 커밋id                // 과거의 특정 commit과 현재 파일 비교
            git diff 커밋id1 커밋id2        // 과거의 특정 commit 2개 차이점 비교
            git difftool                  // 비주얼적으로 좋음
            git graph와 같은 부가기능 사용하면 더 편리함


- 4강
    - branch
        프로젝트의 복사본 만들어서 branch에 먼저 개발

    - branch 생성
        - git branch 브랜치이름
            프로젝트 사본 생성
        
        - git switch 브랜치이름
            git branch로 만든 브랜치로 이동
            이동하면 여기서 개발 및 commit 마음대로 할 수 있음
        
        - git log --graph --oneline --all
            branch와 commit 내역을 한눈에 그래프로 볼 수 있음

        - 브랜치 병합
            git switch main
            git merge 브랜치이름

        - branch conflict
            master 브랜치와 새로운 branch에서 같은 파일과 같은 줄 수정하면 merge conflict 발생
            어떤 파일 남길지 결정했으면
            git add 파일명
            git commit -m '메세지'
